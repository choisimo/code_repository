# 성능 테스트 및 분석에 활용할 수 있는 데이터

- **Flow ID**: 각 트래픽 흐름의 고유 식별자입니다. 이를 통해 여러 흐름 간의 차이점을 분석하거나 특정 흐름을 추적할 수 있습니다.
- **Source IP & Destination IP**: 송수신지 IP 주소는 트래픽이 어디서 시작되고 어디로 도착하는지 파악할 수 있게 도와줍니다. 네트워크 문제 분석 시 중요한 요소입니다.
- **Source Port & Destination Port**: 어떤 포트를 통해 통신이 이루어졌는지 알 수 있습니다. 특정 서비스에 대한 트래픽을 분석할 때 유용합니다.
- **Protocol**: TCP, UDP와 같은 전송 계층 프로토콜을 식별하여 트래픽 유형을 분석할 수 있습니다.
- **Flow Duration**: 트래픽 흐름의 지속 시간을 통해 네트워크 성능 및 효율성을 측정할 수 있습니다.
- **Total Forward Packets & Total Backward Packets**: 송수신된 패킷의 총 개수를 통해 각 연결의 규모를 측정할 수 있습니다.
- **Flow Bytes per Second**: 초당 전송된 데이터의 양은 대역폭을 얼마나 차지했는지 측정하는 중요한 성능 지표입니다.
- **Flow Packets per Second**: 초당 처리된 패킷의 개수를 통해 트래픽의 처리 속도를 분석할 수 있습니다.
- **Flow IAT Mean**: 패킷 간의 평균 간격을 통해 트래픽의 밀도를 파악할 수 있습니다.
- **Label**: 이 흐름이 정상적인지(BENIGN) 또는 악성(MALICIOUS)인지를 나타내어 보안 분석에 사용할 수 있습니다.
- **Protocol Name**: HTTP_PROXY와 같은 프로토콜 이름을 통해 어떤 애플리케이션 계층 프로토콜이 사용되었는지 파악할 수 있습니다.

## 성능 테스트에서 활용 방안

- **대역폭 측정**: Flow Bytes per Second 데이터를 사용하여 네트워크가 얼마나 많은 데이터를 처리할 수 있는지 측정하고, 여러 연결이 발생할 때의 처리량을 분석할 수 있습니다.
- **트래픽 밀도 분석**: Flow Packets per Second 및 Flow IAT Mean을 사용하여 네트워크 트래픽의 혼잡도 및 패킷 간 간격을 분석할 수 있습니다.
- **트래픽 이상 탐지**: Label 필드를 통해 정상(BENIGN)과 비정상(악성) 트래픽을 구분하여 네트워크 보안 문제를 탐지할 수 있습니다.
- **프로토콜 분석**: Protocol Name을 사용하여 HTTP, FTP, DNS 등 다양한 애플리케이션 계층 프로토콜의 트래픽 패턴을 분석할 수 있습니다.

이 데이터를 기반으로 다양한 성능 테스트 및 네트워크 분석을 수행할 수 있으며, 필요한 경우 추가적인 필드를 포함시켜 더 심도 있는 분석도 가능합니다.
